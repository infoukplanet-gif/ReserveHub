# 機能仕様書

## 1. 予約管理

### 1.1 メニュー管理

#### メニュー構造
```
メニュー (Menu)
├── 基本情報
│   ├── メニュー名
│   ├── 説明文
│   ├── カテゴリ
│   ├── 所要時間（分）
│   ├── バッファ時間（分）— 前後の準備時間
│   └── 表示順
│
├── 料金ルール (PricingRule)
│   ├── ベース料金（デフォルト価格）
│   ├── 曜日別ルール
│   │   ├── 平日料金
│   │   ├── 土曜料金
│   │   └── 日祝料金
│   ├── 時間帯別ルール
│   │   ├── 時間帯名（モーニング/デイ/ナイト等、自由設定）
│   │   ├── 開始時刻
│   │   ├── 終了時刻
│   │   └── 料金
│   └── 曜日×時間帯の組み合わせルール
│       └── 例: 平日ナイト=8,000円、休日デイ=12,000円
│
├── オプション (MenuOption)
│   ├── オプション名
│   ├── 追加料金
│   ├── 追加時間（分）
│   ├── 複数選択可否
│   └── 必須/任意
│
└── 対応スタッフ
    └── スタッフID[]

```

#### 料金計算ロジック
```
最終料金 = ベース料金
         → 曜日ルールがあれば上書き
         → 時間帯ルールがあれば上書き
         → 曜日×時間帯ルールがあれば上書き（最優先）
         + 指名料（スタッフに設定されている場合）
         + オプション料金の合計
```

**優先順位**: 曜日×時間帯 > 時間帯 > 曜日 > ベース料金

#### 所要時間計算ロジック
```
合計所要時間 = メニュー所要時間
             + 選択オプションの追加時間合計
             + バッファ時間
```

### 1.2 予約フロー（顧客側）

```
Step 1: メニュー選択
        └── カテゴリ別にメニュー一覧を表示
        └── 各メニューに「○○円〜」と最安値を表示

Step 2: オプション選択
        └── 選択メニューに紐づくオプションを表示
        └── 料金リアルタイム更新

Step 3: スタッフ選択（任意）
        └── 「指名なし」選択可
        └── 指名料が加算される場合は表示
        └── スタッフごとの空き状況を表示

Step 4: 日時選択
        └── カレンダー表示
        └── 選択日の空き時間枠を表示
        └── 曜日/時間帯による料金の違いを明示
        └── 合計金額をリアルタイム表示

Step 5: 顧客情報入力
        └── 名前、電話番号、メールアドレス
        └── リピーターはログインで自動入力
        └── 回数券保有者は回数券適用の選択

Step 6: 確認・予約確定
        └── 内訳明細表示（メニュー+オプション+指名料）
        └── 予約確定 → 確認メール送信
```

### 1.3 予約管理（事業者側）

#### 予約表示
- **日表示**: タイムライン形式（スタッフ別列）
- **週表示**: スタッフ別の週間カレンダー
- **月表示**: 日ごとの予約件数サマリー
- **リスト表示**: 予約一覧（フィルタ・検索可能）

#### 予約操作
- 手動予約追加（電話予約等）
- 予約変更（日時、メニュー、スタッフ）
- 予約キャンセル（キャンセル理由の記録）
- ノーショー記録
- ブロック枠の設定（休憩、会議等）

#### 営業時間設定
- 曜日ごとの営業時間（開始/終了）
- 臨時休業日の設定
- 特別営業日の設定（祝日に営業等）
- 受付締切（○時間前まで）
- キャンセル締切（○時間前まで）

### 1.4 通知

| タイミング | 対象 | 内容 |
|---|---|---|
| 予約確定時 | 顧客 | 予約内容の確認 |
| 予約確定時 | スタッフ | 新規予約の通知 |
| 前日リマインド | 顧客 | 明日の予約リマインド |
| 当日リマインド | 顧客 | 本日の予約リマインド（設定可） |
| 予約変更時 | 顧客+スタッフ | 変更内容の通知 |
| キャンセル時 | 顧客+スタッフ | キャンセル確認 |
| 回数券残り少 | 顧客 | 残回数が○回以下で通知 |

## 2. 顧客管理

### 2.1 顧客情報

```
顧客 (Customer)
├── 基本情報
│   ├── 氏名
│   ├── フリガナ
│   ├── メールアドレス
│   ├── 電話番号
│   ├── 生年月日（任意）
│   ├── 性別（任意）
│   └── 登録日
│
├── タグ（複数可）
│   └── 例: VIP, 回数券保有, 要注意, 紹介客
│
├── カルテ
│   ├── カルテ項目（事業者がカスタム定義）
│   │   ├── テキスト項目
│   │   ├── 選択項目
│   │   ├── 数値項目
│   │   └── 画像項目
│   └── 来店ごとの記録
│
├── 来店履歴（自動記録）
│   ├── 日時
│   ├── メニュー
│   ├── 担当スタッフ
│   ├── 金額
│   └── メモ
│
└── 集計
    ├── 累計来店回数
    ├── 累計売上
    ├── 最終来店日
    ├── 平均来店間隔
    └── よく利用するメニュー
```

### 2.2 カルテ機能

#### カルテテンプレート
事業者が自由にカルテの項目を定義できる:

```
例: 整体院のカルテ
├── 主訴（テキスト）
├── 痛みの部位（選択: 首/肩/腰/膝/...）
├── 痛みレベル（数値: 1-10）
├── 施術内容（テキスト）
├── 施術後の状態（テキスト）
└── 施術写真（画像）

例: パーソナルジムのカルテ
├── 体重（数値）
├── 体脂肪率（数値）
├── トレーニング内容（テキスト）
├── セット数/回数（テキスト）
├── 次回の目標（テキスト）
└── ボディ写真（画像）
```

## 3. 回数券・チケット管理

### 3.1 回数券の構造

```
回数券テンプレート (TicketTemplate)
├── 回数券名（例: 60分コース10回券）
├── 対象メニュー（複数可）
├── 回数
├── 販売価格
├── 有効期限（購入日から○ヶ月）
├── 1回あたりの消化ルール
│   └── 対象: メニュー本体のみ（オプション・指名料は別会計）
└── 販売ステータス（販売中/販売停止）

購入済み回数券 (PurchasedTicket)
├── 顧客
├── 回数券テンプレート
├── 購入日
├── 有効期限
├── 残回数
├── 消化履歴[]
│   ├── 消化日
│   ├── 予約ID
│   └── 担当スタッフ
└── ステータス（有効/期限切れ/使い切り）
```

### 3.2 回数券と予約の連携

```
予約時:
1. 顧客が回数券を保有しているか自動チェック
2. 対象メニューの回数券があれば「回数券を使う」選択肢を表示
3. 回数券使用時:
   - メニュー本体 → 回数券から1回消化
   - オプション料金 → 別途請求
   - 指名料 → 別途請求
4. 残回数を更新
5. 残回数が少ない場合（設定値以下）→ 通知
```

## 4. ホームページ機能

### 4.1 ページ構成

```
事業者のホームページ
├── トップページ
│   ├── ヒーロー画像/動画
│   ├── キャッチコピー
│   ├── 特徴・強み（3-4項目）
│   └── CTAボタン（予約する）
│
├── メニュー・料金ページ
│   ├── カテゴリ別メニュー一覧
│   ├── 料金表（料金ルールから自動生成）
│   ├── オプション一覧
│   └── 回数券一覧
│
├── スタッフ紹介ページ
│   ├── 写真
│   ├── 名前・肩書き
│   ├── 自己紹介
│   └── 得意メニュー
│
├── アクセスページ
│   ├── 住所
│   ├── Google Map埋め込み
│   ├── 営業時間
│   └── 電話番号
│
├── お知らせ/ブログ
│   ├── 記事一覧
│   ├── 記事詳細
│   └── カテゴリ分け
│
└── 予約ページ
    └── 予約フローへ遷移
```

### 4.2 テンプレート・カスタマイズ

- テンプレート数: 初期5種類（シンプル/モダン/ナチュラル/クール/和風）
- カラーテーマ変更
- ロゴ・ファビコン設定
- フォント選択
- セクションの表示/非表示・並び替え
- カスタムCSS（プロプラン以上）

### 4.3 SEO・OGP

- title/description/keywords設定
- OGP画像設定
- 構造化データ（LocalBusiness）自動出力
- sitemap.xml自動生成
- Google Analytics埋め込み
