# 要件定義書

## プロダクト概要

**プロダクト名**: （仮）ReserveHub
**コンセプト**: 予約管理 × ホームページ × 顧客管理を一体化した、中小事業者向けWebアプリ
**差別化**: 料金設定の圧倒的な柔軟性（平日/休日料金、オプション、指名料を1メニューで管理）

## ビジネス目標

- 法人向けWebアプリを入口に、御用聞きで案件を広げる
- 月額5,000〜15,000円のSaaS収益 + カスタム開発案件
- 初期ターゲット: 業種問わず（市場が特化先を教えてくれる戦略）

## ターゲットユーザー

### プライマリ
- 5-30人規模の中小事業者（サロン、ジム、スクール、クリニック等）
- IT担当がいない or 兼任
- STORES予約/RESERVA/SelectTypeに不満を持っている層

### セカンダリ
- 個人事業主（整体、カメラマン等）— 無料プランで取り込み

## 機能要件

### P0（MVP必須）

#### 1. 予約管理
- [ ] カレンダーベースの予約表示・管理
- [ ] メニュー/コース作成
- [ ] **柔軟な料金設定**
  - 曜日別料金（平日/土日祝）
  - 時間帯別料金（モーニング/デイ/ナイト）
  - 料金ルールの組み合わせ（曜日×時間帯）
- [ ] **オプション料金**
  - メニューにトッピング式で追加
  - 複数オプション同時選択可
  - オプションごとに所要時間の加算設定
- [ ] **スタッフ指名**
  - スタッフ別のスケジュール管理
  - 指名料の自動加算
  - スタッフごとの対応メニュー設定
- [ ] 予約枠の自動計算（メニュー時間+オプション時間+バッファ）
- [ ] 予約確認メール自動送信
- [ ] リマインドメール（前日/当日）
- [ ] キャンセル・変更機能

#### 2. 顧客管理（簡易CRM）
- [ ] 顧客台帳（自動登録）

- [ ] 顧客メモ/カルテ機能
- [ ] タグ・セグメント分け
- [ ] 顧客別の売上集計

- [ ] 回数券購入履歴

#### 4. ホームページ機能

- [ ] スタッフ紹介ページ
- [ ] 予約ボタンの埋め込み
- [ ] お知らせ/ブログ機能
- [ ] 独自ドメイン対応
- [ ] OGP/SEO設定
- [ ] レスポンシブ対応

#### 5. 管理ダッシュボード

## 非機能要件

### パフォーマンス
- ページ読み込み: 2秒以内
- 予約操作のレスポンス: 1秒以内
- 同時接続: 100ユーザー以上

### セキュリティ
- HTTPS必須
- 認証: メール+パスワード、Google OAuth
- 個人情報の暗号化
- CSRF/XSS対策
- レート制限

### 可用性
- 稼働率: 99.5%以上
- 自動バックアップ（日次）

### 対応環境
- モダンブラウザ（Chrome, Safari, Firefox, Edge）
- スマートフォン/タブレット/PC
