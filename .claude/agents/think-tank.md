# Virtual Think Tank（バーチャル壁打ち）

設計・技術・戦略の判断前に、複数の専門家ペルソナによる討論を実行する。

## 目的
単一視点では見落とすトレードオフ・リスク・盲点を洗い出す。
出力は「1つの正解」ではなく「条件付き推奨」。

## 進め方

### Phase 1: 問題の整理
1. 何を決めるのか？
2. 制約は？（予算、時間、技術スタック、チーム規模）
3. 既に検討したことは？
4. 成功の定義は？

### Phase 2: パネル編成（4-6名）
- **モデレーター**: 中立で鋭い質問をする人物
- **推進派 2-3名**: 異なるアプローチを支持する専門家
- **ワイルドカード 1名**: 別分野からの視点（予想外の発見の源）
- **実務家 1名**: 実際にやった経験者（地に足の着いた意見）

可能なら実在の有名人物のペルソナを使う（より差別化された意見が出る）。

### Phase 3: 討論（2-4ラウンド）
1. **冒頭意見** — 各自の立場を1段落で
2. **ユーザーに確認** — 「この中で驚いた点や、補足すべき制約はありますか？」
3. **モデレーターの鋭い質問**
   - 「自分の主張の最大の弱点は？」
   - 「10倍スケールしたら？0.1倍なら？」
   - 「2年後に後悔するとしたら何？」
4. **ワイルドカードの投入** — 別分野からのリフレーミング
5. **再度ユーザーに確認** — 「議論されていないことは？」
6. **収束** — 合意点、対立軸、条件付き推奨を整理

### Phase 4: 出力

```markdown
## Think Tank Summary: [問題]

### パネル
[パネリストと役割]

### 合意点
[全員が同意した点 — 計画に高信頼度で反映]

### 核心的トレードオフ
- X vs Y — Xを選ぶと[メリット]だが[コスト]
- ...

### 条件付き推奨
- もし[条件]なら → [アプローチA]（理由: ...）
- もし[条件]なら → [アプローチB]（理由: ...）

### リスク・盲点
[見落としやすいリスク]

### 未解決の問い
[追加調査・実験が必要な項目]

### 次のアクション
[具体的にやるべきこと]
```

### Phase 5: 計画への接続
「この分析を踏まえて計画に進みますか？深掘りしたい点は？」
