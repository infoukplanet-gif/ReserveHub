# 振り返り（Retrospective）

## 手順

### 1. 会話から抽出

| カテゴリ | 抽出するもの |
|---|---|
| **決定事項** | 仕様・設計・方針で決まったこと |
| **新しい知見** | 競合情報、ユーザーの声、技術的発見 |
| **課題・TODO** | 次回以降に対応が必要なこと |
| **失敗・修正** | うまくいかなかったこと、バグの原因パターン |
| **開発ルール** | 新しく生まれたルールや規約 |

### 2. docs/changelog.md に追記

```markdown
## YYYY-MM-DD — セッションタイトル

### 決定事項
- ...

### 新しい知見
- ...

### 課題・TODO
- [ ] ...

### 失敗・修正から学んだこと
- ...

### 開発ルールの追加・変更
- ...
```

### 3. 仕様書の反映判断
docs/ 配下で更新すべき箇所を特定し、リストアップする。

### 4. Agent/Skill 更新提案

以下に該当する場合、更新を提案:
- 新しい開発ルール → Agent のルールセクションに追加
- バグパターン発見 → reviewer/qa のチェック項目に追加
- 新しい技術導入 → 該当 Agent の技術スタックに反映
- ワークフロー改善 → Skill の手順を更新
- 不要になったルール → 削除提案

```
## 更新提案
- [更新] reviewer.md — ○○チェック項目を追加（理由: ○○）
- [新規] skills/xxx/ — ○○のために必要
- [削除] ○○ — 不要になったため
```

### 5. ユーザーに確認
提案を提示し、承認されたら /update-agents を実行。
