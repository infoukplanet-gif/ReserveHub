# ReserveHub デザイン方針 — 取り入れるもの・排除するもの

## 排除するもの（禁止リスト）

| 禁止 | 理由 |
|---|---|
| グラデーション（全種類） | 好みに合わない。ソリッドカラーで統一 |
| Aurora UI / グロウ / ネオン | AI臭い。安っぽく見える |
| AIイラスト / ストック画像 | AI感が出る。使うなら実写のみ |
| 過度なアニメーション（Blob、パーティクル、3D） | うるさい。業務ツールに不要 |
| pill型ボタン（角丸9999px） | Badge以外禁止。ボタンは角丸8px |
| ダークモードのランディングページ | SaaS管理ツールに合わない |

## 取り入れるもの

### 1. Bento Grid（ダッシュボード）
- 不均一サイズのカードグリッド
- 情報の重要度をサイズで表現
- 適用: B-1 ダッシュボード

### 2. Micro-interactions（全画面）
- 全て00_design-system.mdのアニメーション仕様表に定義済み
- 装飾ではなく状態変化の通知として使う
- 特に重要: 料金のリアルタイム更新、ステッパー進行

### 3. 構造的な美しさ（Notion / Linear 方向）
- 白+グレー階調を基本に、primaryの青をアクセント
- 余白を十分に取る（詰め込まない）
- 要素の整列を厳密に（グリッドに乗せる）
- タイポグラフィの階層で情報構造を伝える

### 4. 実用的なインタラクション
- ホバーで情報を追加表示（Tooltip）
- 選択状態を明確にする（border + background変化）
- 操作結果を即座にフィードバック（Toast）
- ドラッグ&ドロップ（メニュー並び替え、セクション並び替え）

## 参考プロダクト

| プロダクト | 参考にする点 |
|---|---|
| **Notion** | クリーンな白基調、タイポの階層、余白の使い方 |
| **Linear** | ステータス管理UI、キーボード操作、レスポンスの速さ |
| **Stripe Dashboard** | データ表示の明快さ、テーブルUI、フィルタ |
| **Cal.com** | 予約UIの参考、ステッパー、時間枠選択 |
| **Vercel Dashboard** | サイドバー、デプロイ状態表示、ミニマルさ |

**共通点: フラット、ソリッド、クリーン、構造で魅せる。装飾に頼らない。**
