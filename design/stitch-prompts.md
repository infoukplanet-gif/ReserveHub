# Stitch プロンプト集 — コピペ用

各プロンプトをそのままStitchのチャットに貼り付けてください。
最初に「0. デザインシステム」を投げてから、各画面を順番に投げてください。

---

## 0. デザインシステム（最初に必ず投げる）

```
これから予約管理SaaS「ReserveHub」のUIを設計します。全画面で以下のデザインシステムに従ってください。

プロダクト: 中小事業者向けの予約管理Webアプリ（サロン、ジム、クリニック等が対象）
言語: 日本語UI
スタイル: Notion、Linear、Stripe Dashboardのようなクリーン・フラット・ソリッド
フレームワーク: Tailwind CSS

【絶対禁止】
- グラデーション（一切使わない。ボタンも背景も全てソリッドカラー）
- AI生成イラスト、ストック画像
- グロウ、ネオン、発光エフェクト
- pill型ボタン（Badgeのみ丸め可）
- ダーク背景のランディングページ
- 装飾のためだけのアニメーション

【カラー】
- Primary: #2563EB（1画面に1つだけのメインアクション）
- Primary Hover: #1D4ED8
- 選択中背景: #EFF6FF
- 成功/完了: #16A34A
- 警告: #F59E0B
- エラー/削除: #DC2626
- ページ背景: #F8FAFC
- カード背景: #FFFFFF
- ボーダー: #E2E8F0
- テキスト: #0F172A
- サブテキスト: #64748B

【タイポグラフィ】
- フォント: Inter + Noto Sans JP
- H1: 28px Bold（1ページに1つ）
- H2: 22px SemiBold
- H3: 18px SemiBold
- 本文: 15px Regular
- 金額大: 28px Bold、金額中: 18px SemiBold

【コンポーネント】
- ボタン角丸: 8px、カード角丸: 12px
- カード: 白背景、1pxボーダー#E2E8F0、shadow-sm
- インプット: 高さ40px、角丸8px
- ステータスBadge: pill型、確定=青、完了=緑、キャンセル=赤

この設定を記憶して、以降の全画面に適用してください。
```

## 1. 予約フロー — Step4 日時選択（最重要画面）

```
ReserveHubの予約フロー画面を作ってください。6ステップのStep4「日時選択」画面です。Desktop版とMobile版の両方。

【ステッパー】画面上部に水平ステッパー
- 6ステップ: ①メニュー → ②オプション → ③スタッフ → ④日時 → ⑤情報 → ⑥確認
- ①②③は完了（青丸+チェック）、④が現在（青丸+番号、大きめ）、⑤⑥は未到達（灰色丸）
- ステップ間は線で接続

【メインコンテンツ — 左側】
- 見出し: 「日時を選ぶ」（H2）
- カレンダー: 月表示（◀ 2026年4月 ▶）
  - 土曜は青文字、日祝は赤文字
  - 選択日は#2563EB背景白文字
  - 休業日は×マーク灰色
  - 過去日は灰色選択不可
- カレンダーの下に「4月5日(土)の空き時間」見出し
- 空き時間枠リスト:
  - 各行: 時刻 + 料金 + [選択]ボタン
  - 10:00 ¥10,000、11:30 ¥10,000、13:00 ¥10,000、14:30 ¥10,000、16:00 ¥10,000
  - 料金の横に小さく「休日料金」ラベル（Badge）

【料金サイドバー — 右側（Desktopのみ）】
- 白カード、1pxボーダー、角丸12px
- 見出し: 「予約内容」

  - 60分ボディケアコース  ¥10,000
  - ヘッドスパ追加  +¥1,500
  - 指名(山田花子)  +¥500

- サイドバーなし。代わりに画面下部に固定バー:
  - 「合計 ¥12,000 [次へ進む]」
  - 背景: rgba(255,255,255,0.9) + backdrop-filter: blur(12px) + border-top 1px

サンプル店舗: リラクゼーションサロン BLOOM
```
